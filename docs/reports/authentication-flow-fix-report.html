1. Issue Summary

Symptoms

Error Message:

{
    "error": "Cannot destructure property 'user' of 'auth1' as it is undefined.",
    "status": 500
}

User Impact

  • User could sign in via Clerk successfully
  • After sign-in, user was redirected back to unified setup page
  • Admin dashboard was inaccessible
  • All authenticated users were affected

2. Root Cause Analysis

The Problem

The authentication system had a null safety issue in the actions route handler.

Location: src/app/api/resources/[resource]/actions/route.ts (Line 58)

Before Fix:

// Check action-specific permissions const { user } = auth; // ❌ CRASHES if auth is null/undefined! const allowedRoles = action.allowedRoles; if (allowedRoles && user && !allowedRoles.includes(user.type as any)) { return errorResponse("Unauthorized for this action", 403); }

Why This Happened

The code assumed auth would always be defined, but the createApiRoute wrapper can pass null for the auth parameter when:

  1. Authentication check fails
  2. Session is invalid
  3. User is not found in database

3. The Fix

Code Change

After Fix:

// Check action-specific permissions // auth comes from createApiRoute wrapper as { userId, user } if (!auth) { return errorResponse("Unauthorized", 401); // ✅ Safe check first! } const { user } = auth; // ✅ Now safe to destructure const allowedRoles = action.allowedRoles; if (allowedRoles && user && !allowedRoles.includes(user.type as any)) { return errorResponse("Unauthorized for this action", 403); }

What Changed

  1. Added null check before destructuring auth
  2. Early return with 401 if auth is missing
  3. Added comment explaining auth structure

6. Data Flow Details

Auth Object Structure

Important: There are TWO different user IDs in the system:

  • auth.userId = Database ID (users.id) - e.g., "user-platform-admin-001"
  • auth.user.clerkUserId = Clerk ID - e.g., "user_3AIG..."
Property Type Description
auth.userId string Database primary key (users.id)
auth.user.id string Same as auth.userId
auth.user.clerkUserId string Clerk's user ID
auth.user.type string User role (admin, teacher, etc.)
auth.user.onboardingStatus string completed, pending_approval, etc.
auth.user.schoolId string | null School association

7. Key Takeaways

✅ Best Practices Applied

  1. Null Safety First: Always check for null/undefined before destructuring
  2. Early Returns: Fail fast with clear error messages
  3. Code Comments: Document expected data structures
  4. Systematic Debugging: Trace the full auth flow end-to-end

⚠️ Common Pitfalls to Avoid

  • Don't assume wrapper functions always return populated objects
  • Don't destructure without null checks
  • Don't modify middleware when fixing API issues
  • Don't confuse Clerk userId with Database userId
